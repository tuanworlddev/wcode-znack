## <span id="page-3-0"></span>**1. Описание и назначение функциональности**

Данная инструкция описывает пользовательский интерфейс Личного кабинета Реестра партнёров и интеграторов, его функциональные возможности, а также основные вопросы, возникающие при работе с Реестром интеграторов и партнёров на практике.

## <span id="page-4-0"></span>**2. Начало работы**

Для начала работы следует выполнить следующие действия:

• перейти в Реестр интеграторов и партнёров по ссылке [https://registry.intuot.crpt.ru;](https://registry.intuot.crpt.ru)

![](Инструкция_по_работе_с_РИ-v32.0-30.04.2026-at-16-38-08_images/_page_4_Picture_3.jpeg)

*Рисунок 1. Страница входа в реестр интеграторов и партнёров*

- нажать кнопку **[ Зарегистрироваться ]**, заполнить и отправить на проверку формуляр регистрации компании (см. раздел «[Формирование заявки на регистрацию»](#page-6-0));
- после одобрения заявки Оператором, получить данные для входа в систему в письме, направленном на адрес электронной почты, указанный в заявке;
  - В случае отклонения заявки на адрес электронной почты будет отправлено уведомление с причиной отклонения заявки. Для устранения указанной причины подать заявку на регистрацию повторно, указав корректные сведения.
- авторизоваться в системе, ознакомиться и принять условия использования (см. раздел [«Авторизация и восстановление доступа»](#page-17-0));
- проверить и при необходимости отредактировать данные профиля организации (см. разделы [«Редактирование данных об организации](#page-22-0)»);
- выполнить необходимые действия для перевода профиля организации в статус **«Активен»**.
  - В зависимости от типа организации, присвоенного организации (подробнее см. раздел [«Таблица соответствия типов организации видам деятельности](#page-11-0)»), предусмотрены следующие сценарии перехода профиля в статус **«Активен»**:
    - для типа организации **«Участник оборота товаров»** статус **«Активен»** проставляется автоматически;
    - для типов организаций **«Партнёр»**, **«Интегратор»**, **«Самозанятый разработчик»**, **«Сервис-провайдер»** смена статуса возможна только после прохождения проверки организации со стороны Службы Безопасности.

В случае отклонения заявки со стороны Службы Безопасности на адрес электронной почты будет отправлено соответствующее уведомление. При возникновении вопросов обратиться в службу технической поддержки по адресу, указанному в уведомлении.

- для типа организации **«Интегратор»**, **«Самозанятый разработчик»** и **«Участник оборота товаров»** добавить продукты («[Управление интеграционными и партнёрскими продуктами](#page-26-0)») и подписать связанные документы. Перечень документов будет направлен на адрес электронной почты при переходе профиля организации в статус **«Активен»** (подробнее см. в разделе «[Таблица соответствия обязательности документов и типов организаций»](#page-98-0));
- для типа организации **«Сервис-провайдер»** заполнить анкету Сервис-провайдера (см. раздел [«Заполнение анкеты для подключения Сервис-провайдера](#page-105-0)»), дождаться перехода анкеты в статус **«Подтверждена»** после окончания проверки данных Оператором, создать и подписать заявление на договор ПАК (см. раздел «[Подача заявления на договор ПАК](#page-107-0)»), а также автоматически сгенерированный договор ПАК. После успешного завершения обработки документов организация с типом **«Сервис-провайдер»** будет добавлена в перечень сервиспровайдеров в пользовательский интерфейс СУЗ;
- для организаций с видами деятельности **«Оператор фискальных данных»** и **«Оператор электронного документооборота»** доступна функциональность выгрузки статистических данных по ОФД и ЭДО соответственно (подробнее см. в разделе «Статистика по ОФД и ЭДО»).

Подробнее об основных функциональных возможностях Личного кабинета Реестра интеграторов и партнёров см. в разделе «[Основные функциональные возможности Реестра](#page-20-0) [интеграторов и партнёров»](#page-20-0).

## <span id="page-6-0"></span>**3. Формирование заявки на регистрацию**

Для регистрации в Реестре партнёров и интеграторов перейти по ссылке [https://registry.intuot.crpt.ru,](https://registry.intuot.crpt.ru) нажать кнопку **[ Зарегистрироваться ]** и заполнить поля формуляра регистрации компании.

![](Инструкция_по_работе_с_РИ-v32.0-30.04.2026-at-16-38-08_images/_page_6_Picture_2.jpeg)

*Рисунок 2. Страница входа в систему*

Скачивание инструкции по работе в Реестре интеграторов и партнёров доступно по ссылке **«Скачать инструкцию для регистрации и работы в реестре»**.

В формуляре регистрации компании указать следующие сведения:

• **«Страна регистрации бизнеса»** (по умолчанию, **«Россия»**);

При выборе значения **«Физ лицо»** в поле **«Форма регистрации бизнеса»**, доступно только значение **«Россия»**.

![](Инструкция_по_работе_с_РИ-v32.0-30.04.2026-at-16-38-08_images/_page_7_Picture_0.jpeg)

*Рисунок 3. Формуляр регистрации компании*

• **Основной язык** (по умолчанию, **«Русский язык»**)

Возможные значения:

- **«Русский язык»**;
- **«Английский язык»**.

При выборе страны, отличной от **«Россия»**, автоматически устанавливается **«Английский язык»** (при этом выбор значения **«Русский язык»** остаётся доступным).

• **«Форма регистрации бизнеса»** (передвинуть соответствующий переключатель в активное положение). В зависимости от выбранного значения меняется состав полей формуляра регистрации компании.

Выбранное значение является неизменяемым и не может быть отредактировано в ходе дальнейшей работы в системе.

#### Возможные значения:

◦ **«Юр лицо»** — юридическое лицо.

При рассмотрении заявки Оператором проверяется, что юридическое лицо зарегистрировано на территории страны, указанной в поле **«Страна регистрации бизнеса»**.

◦ **«ИП»** — индивидуальный предприниматель.

К регистрации допускаются индивидуальные предприниматели, у которых вид гражданства отличается от «РФ», при условии, что страной регистрации бизнеса для таких индивидуальных предпринимателей является РФ.

◦ **«Физ лицо»** — физическое лицо. Значение выбирается в случае, если указанное физическое лицо является разработчиком ПО для маркировки и зарегистрировано в качестве плательщика налога на профессиональный доход (самозанятого).

Данное значение соответствует типу организации **«Самозанятый разработчик»**, включающий следующие возможные роли для физических лиц:

- интегратор программных и аппаратных решений;
- разработчик кассового программного обеспечения;
- разработчик товаро-учётного программного обеспечения;
- разработчик систем планирования ресурсов и процессов (ERP системы);
- разработчик программно-аппаратных комплексов криптографии и средств информационной защиты;
- разработчик ПО Track&Trace (Маркировки и прослеживаемости);
- консалтинговая деятельность;
- разработка ПО для сервис-провайдеров.

При выборе значения **«Физ лицо»**, в поле **«Страна регистрации бизнеса»** доступно только значение **«Россия»**.

При выборе значения **«Физ лицо»**, в поле **«3. Вид деятельности»** автоматически активируется значение **«Самозанятый разработчик»**, все остальные значения блокируются. При этом в нижней части формы появляется кнопка **[ Скачать согласие на обработку персональных данных ]**, при нажатии которой доступно скачивание файла «Согласие на обработку персональных данных», автоматически заполненного на основании данных заявки.

- оказание услуг доставки товаров потребителям;
- оказание услуг почтовой связи в отношении почтовых отправлений с наложенным платежом при продаже товаров по образцам или дистанционным способом;
- компания выступает платежным агентом, не осуществляющим оборот маркированного товара;

# <span id="page-20-0"></span>**5. Основные функциональные возможности Реестра интеграторов и партнёров**

В Реестре интеграторов и партнёров доступны следующие функциональные возможности:

- просмотр и редактирование данных об организации (подробнее см. в разделе [«Редактирование данных об организации](#page-22-0)»);
- просмотр, добавление новых продуктов и формирование заявок на их тестирование (подробнее см. в разделе «[Управление интеграционными и партнёрскими продуктами](#page-26-0)»). Просмотр [статистики](#page-110-0) использования интеграционных продуктов;
- просмотр и добавление данных о пользователях системы (подробнее см. в разделе [«Редактирование данных о пользователях»](#page-94-0));
- просмотр и подписание документов (подробнее см. в разделе «[Работа с документами](#page-97-0)»);
- просмотр уведомлений от Оператора (подробнее см. в разделе [«Просмотр уведомлений](#page-109-0)»);
- работа с задачами от Оператора (подробнее см. в разделе [«Работа с задачами от Оператора](#page-112-0)»);
- просмотр справочной информации о нормативно-правовых актах, регламентирующих процессы маркировки, и их изменениях (подробнее см. в разделе «[Сведения о нормативно](#page-121-0)[правовых актах»](#page-121-0));
- просмотр новостей, касающихся Реестра интеграторов и партнёров (подробнее см. в разделе [«Просмотр новостей»](#page-123-0)).

## <span id="page-22-0"></span>**5.2. Редактирование данных об организации**

Для внесения изменений в регистрационные данные, в **«Главном меню»** перейти в раздел **«Моя организация» › «Организация»** и нажать кнопку в карточке организации.

В этом режиме доступны для изменения следующие поля:

- **«КПП»**;
- **«Сокращенное название»** (после перехода профиля организации в статус **«Активен»**, значение заблокировано для изменения);
- **«Длинное название»** (после перехода профиля организации в статус **«Активен»**, значение заблокировано для изменения);
- **«Основной email компании»**;
- **«Телефон»**;
- **«WEB сайт»**;
- **«Профиль»**;
- **«Наличие партнёрской сети»** если применимо, активировать перемещением переключателя в правое положение;

• **«ФИО доверенного лица»**.

Если заполнено, подписать доверенность в разделе **«Документы»**, после добавления соответствующего документа Оператором.

Для корректировки полей, заблокированных после перехода профиля организации в статус **«Активен»**, обратиться в службу технической поддержки по адресу [support@crpt.ru](mailto:support@crpt.ru).

Для редактирования банковских реквизитов см. раздел «[Редактирование банковских](#page-23-0) [реквизитов](#page-23-0)». Банковские реквизиты используются для заполнения NDA (если указаны).

Перечень значений поля **«Профиль деятельности»** может быть отредактирован только Оператором через заявку от пользователя (подробнее см. в разделе «[Подача заявки на изменение](#page-25-0) [перечня видов деятельности](#page-25-0)»).

## <span id="page-26-0"></span>**5.3. Управление интеграционными и партнёрскими продуктами**

В Реестре интеграторов и партнёров предусмотрено два вида продуктов:

• партнёрские продукты.

К партнёрским продуктам относятся продукты, которые не имеют взаимодействия с API СУЗ, например, онлайн-кассы, УПД Операторы, ТСД сканеры и т.д. Такие продукты не проходят приёмочные испытания на взаимодействие с компонентами ГИС МТ или ФГИС МДЛП, при этом для них обязательно подписание таких документов как Декларация, NDA и Заявление о вступлении в Меморандум.

Подробнее о партнёрских продуктах см. в разделе «[Партнёрские решения](#page-90-0)».

• продукты для интеграции.

К продуктам для интеграции относится программное обеспечение, которое предназначено для взаимодействия с API СУЗ и требует тестирования со стороны Оператора согласно методике проверки пусконаладочных работ, принятой для API СУЗ. Версии программного обеспечения, прошедшие приёмочные испытания, получают статус **«Допущен в промышленную среду»**, а также уникальный ключ регистрации для работы на промышленном контуре ГИС МТ или ФГИС МДЛП. Доступ к такому программному обеспечению появляется в пользовательском интерфейсе СУЗ в разделе **«Реестр интеграционных решений»** для всех участников оборота товаров (при соответствующих настройках видимости).

При этом также обязательно подписание таких документов как Декларация, NDA и Заявление о вступлении в Меморандум.

Подробнее о продуктах к интеграции см. в разделе «[Продукты к интеграции](#page-27-0)». Подробнее о статистике использования интеграционных продуктов см. в разделе «[Просмотр статистики](#page-110-0) Таблица доступности продуктов по типам организаций представлена ниже.

| Тип организации          | Продукт              | Испытание СУЗ |
|--------------------------|----------------------|---------------|
| Партнёр                  |                      | —             |
| Интегратор               | Партнёрское решение  | —             |
| Сервис-провайдер         |                      | —             |
| Интегратор               |                      | +             |
| Участник оборота товаров | Продукт к интеграции | +             |
| Самозанятый разработчик  |                      | +             |

#### <span id="page-27-0"></span>**5.3.1. Продукты к интеграции**

Работа с продуктами для интеграции в Реестре интеграторов и партнёров доступна организациям с типами организации **«Интегратор»**, **«Участник оборота товаров»** и **«Самозанятый разработчик»**.

Для регистрации продукта к интеграции и получения версией продукта статуса **«Допущен в промышленную среду»**:

- зарегистрировать продукт в Реестре интеграторов и партнёров (см. раздел [«Регистрация](#page-28-0) [продукта к интеграции](#page-28-0)»);
- самостоятельно протестировать продукт согласно методике проверки пусконаладочных работ, принятой для API СУЗ.

Для ознакомления с методикой, в **«Главном меню»** выбрать раздел **«База знаний»**, перейти по ссылке **«Методика проверки пусконаладочных работ RU»** и скачать описание в формате \* .pdf.

- подать заявку на приёмочные испытания в СУЗ и протестировать интеграцию с СУЗ по API:
  - с участием службы технической поддержки Оператора (см. раздел [«Тестирование с](#page-35-0) [участием службы технической поддержки Оператора»](#page-35-0));
  - без участия службы технической поддержки Оператора (см. раздел [«Автотестирование](#page-81-0) [интеграционного решения](#page-81-0)»).

Данная возможность доступна при отсутствии активных заявок на проведение приёмочных испытаний совместно с Оператором;

• при необходимости добавить доверенные ИНН (см. раздел «[Добавление доверенных ИНН»](#page-87-0)).

- «Описание» / «Описание / Состав модели ТС ПИоТ» (отображается, если активировано поле «Направление разработки ТС ПИоТ?») указать описание ПО прямым вводом, достаточное для идентификации его качеств участниками оборота товаров;
- «Телефон службы тех. поддержки» указать значение прямым вводом в свободной форме;
- «**Email тех. поддержки»** указать значение прямым вводом;
- «Облачное решение» если применимо, активировать значение перемещением переключателя вправо.

Активация значения подтверждает, что программное обеспечение является облачным

решением с авторизацией на web-сервисе и возможностью подключения множества участников оборота товаров;

• «Направление разработки ТС ПИоТ?» — если применимо, активировать значение перемещением переключателя вправо.

Активация значения подтверждает, что программное обеспечение является продуктом ТС ПИоТ, а также позволит подать заявку на приёмочные испытания;

• «**Версия**» — указать номер версии программного обеспечения, которая будет протестирована в рамках приёмочных испытаний в СУЗ.

Регистрация новых версий продукта производится в упрощённом режиме. Подробнее см. в разделе «Регистрация новых версий продукта».

• «Видимость» — выбрать значение выпадающего списка («Да» / «Нет»).

Если выбрано «Да», то после успешного прохождения приёмочных испытаний продукт будет отображаться в Реестре протестированных интеграционных решений СУЗ.

Если программное обеспечение разрабатывается для личного (некоммерческого) использования, выбрать **«Нет»**.

- «Выполняемые задачи» привести описание задач, выполняемых продуктом в рамках указанной версии. Данное описание позволяет участникам оборота товаров идентифицировать реализованные функции продукта по конкретной версии.
- нажать кнопку [ Сохранить ] для завершения регистрации продукта;
- заполнить анкету по продукту (подробнее см. в разделе «Заполнение анкеты по продукту»);
- после заполнения анкеты и подтверждения корректности со стороны Оператора, ознакомиться и подписать обязательные документы, размещённые Оператором в разделе «Моя организация» > «Документы» (подробнее см. в разделе «Подписание документов») в соответствии с таблицей, указанной в разделе «Таблица соответствия обязательности документов и типов организаций»:
  - Декларация на продукт;
  - Заявление о вступлении в Меморандум;
  - NDA.

#### <span id="page-30-0"></span>5.3.1.1.1. Перевод продукта к интеграции в статус «Недействителен»

◦ внести результаты прохождения текущего шага тест-кейса в поле **«Фактический результат»**. В поле **«Действие»** дана информация о том, какой тип сведений необходимо указать в поле **«Фактический результат»** (логи API запросов и ответов или иные данные).

Поле **«Фактический результат»** доступно, только если набор тестов находится в статусе **«В работе»**;

◦ нажать кнопку **[ Успешно ]** в случае совпадения фактического результата с ожидаемым или **[ Неуспешно ]** — при расхождении результата.

Кнопки доступны, только если набор тестов находится в статусе **«В работе»**.

При выборе значения **«Неуспешно»** для обязательного шага тестирования статус процесса прохождения тестов изменится на **«Не пройден»**. В остальных случаях статус принимает значение **«Незавершен»**;

◦ перейти к выполнению следующего шага методики тестирования.

Для быстрого перехода между шагами методики тестирования выбрать тест в списке **«Шаги»** в правой части экрана.

Для приостановки выполнения тестирования нажать кнопку **[ Приостановить ]** в правом верхнем углу экрана. В этом случае статус процесса прохождения тестов изменится на **«Отложен»**. Для продолжения тестирования нажать кнопку **[ Продолжить тестирование ]**.

• после завершения всех обязательных шагов методики тестирования нажать кнопку **[ Завершить ]** для отправки результатов тестирования на проверку сотрудниками службы технической поддержки Оператора. При этом набор тестов переходит в статус **«Проверка»**, и на электронную почту контактного лица будет направлено письмо о начале процедуры проверки заявки.

В случае, если тест-кейс требует внесения изменений со стороны контрагента, Оператор может перевести заявку на доработку. При этом набор тестов переходит в статус **«На доработке»** и на электронную почту контактного лица будет направлено письмо о том, что заявка на тестирование отправлена на доработку;

- после успешной проверки корректности всех шагов тест-кейса Оператором, статус заявки изменяется на **«Согласование даты приемки»**. Оператор связывается с контрагентом и согласовывает дату и время итоговой приёмки тест-кейса в режиме видеоконференцсвязи, а заявка переходит в статус **«Приемка»**. На приёмке осуществляется демонстрация следующих шагов проверки:
  - «Отчёт об использовании отправляется на основе отчёта о валидации КМ с оборудования L3 (машинное зрение, сканер и т.д.)» (обязательно);

- «Экспортируются только коды идентификации из ПОСС» (обязательно);
- «Соблюдение правил хранения и использования кодов маркировки» (обязательно);
- «Проверка минимального остатка лицевого счета УОТ» *(опционально)*;
- «Автоматическое пополнение запаса КМ» *(опционально)*;
- «Контроль срока годности кодов маркировки» *(опционально)*.

В случае, если данные шаги требуют доработки или исправления со стороны контрагента, Оператор отправляет заявку на исправление, а набор тестов переходит в статус **«Ожидание исправления»**. После проведения исправлений контрагент переводит заявку в статус **«Согласование даты приемки»**.

После успешного проведения приёмки статус процесса прохождения тестов изменится на **«Пройден»**.

  - **«% Ошибок без дубликатов»** доля документов, обработанных с ошибками (без учёта документов с ошибкой «4 — DuplicateDocument: Документ <идентификатор документа> зарегистрирован ранее»), относительно общего количества поступивших документов;
  - **«Всего ошибок»** количество документов, обработанных с ошибками;
  - **«% ошибок»** доля документов, обработанных с ошибками, относительно общего

количества поступивших документов;

◦ распределение количества документов, обработанных с ошибками, по типам ошибок (описание представленных ошибок см. в [«Описание ошибок обработки документов в](https://честныйзнак.рф/business/doc/?id=Инструкция_по_возникающим_ошибкам.html) [личном кабинете и по API»\)](https://честныйзнак.рф/business/doc/?id=Инструкция_по_возникающим_ошибкам.html).

Информация о доли эмитированных КМ, поступившая от текущего оператора отмечена на гистограмме столбцом красного цвета;
- в табличном виде отображается информация по закрытым квитанциям СУЗ от текущего оператора (элемент 5):
  - **«Товарная группа»** товарная группа, по которой сформирована статистика СУЗ;
  - **«Зарегистрировано заказов КМ»** количество зарегистрированных заказов КМ;
