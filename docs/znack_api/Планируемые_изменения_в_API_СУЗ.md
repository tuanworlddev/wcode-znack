# СУЗ-ОБЛАКО 4.0 АНОНСИРОВАНИЕ ПЛАНИРУЕМЫХ ИЗМЕНЕНИЙ В СУЗ Основные функции Редакция 1.26

Листов 25

### **АННОТАЦИЯ**

В данном документе приведена информация для предварительного информирования участников оборота товаров об изменениях в СУЗ API, планируемых к реализации в следующие 1-2 месяца. Описываемые изменения носят информационный характер и публикуются заранее с целью предоставления возможности УОТ спроектировать проведение интеграционных работ с СУЗ.

### **1. АНОНСИРОВАНИЕ ПЛАНИРУЕМЫХ ИЗМЕНЕНИЙ В API СУЗ**

<span id="page-3-0"></span>В данном разделе приведен перечень планируемых изменений в API СУЗ, находящиеся на стадии анализа.

- 1. Планируется отключение поддержки прикрепленной подписи. Даты отключения будут анонсированы позже.
- 2. Параметр createMethodType в структуре заказа во всех интерфейсах системы (API V3, API ПАК, API ЕАЭС) переводится в статус устаревшего. Параметр будет считается опциональным, валидация его значения снята. Структура заказа будет корректно обрабатываться как с указанием параметра createMethodType, так и без него.
- 3. Для ТГ «Специализированная пищевая продукция и БАД к пище» будет доступна эмиссия кодов маркировки для товаров по новой 4 волне платности с 31 марта 2026г.
- 4. Для товарной группы "Сладости и кондитерские изделия" с 29 апреля 2026 года будет представлена возможность подавать в отчете о нанесении информацию о КМ с вариативным сроком годности в параметре «expDateConditions» - Дата окончания срока годности продукции (срок хранения более 72 часов) с указанием индекса условия хранения. В данном атрибуте указывается набор значений сроков годности. Каждое вложение набора должно включать индекс условия хранения, указанный в НКМТ, и дату окончания срока годности (больше или равный текущей дате) в формате yyyy-MM-dd. Может быть указано несколько сроков годности. Индекс условия хранения в наборе должен быть уникальным в пределах набора значений.
- 5. Для товарной группы «Медицинские изделия» будет проводиться маркировка товара в рамках эксперимента для GTIN 8 группы платности в период с 29.04.2026 до 01.03.2027г.
- 6. Для товарной группы «Молочная продукция» будет доступна эмиссия для пятой группы платности, с 01 июля 2026г доступен тип оплаты – «Оплата по эмиссии».
- 7. Для товарной группы «Пиво, напитки, изготавливаемые на основе пива и слабоалкогольные напитки» вводится требование по заполнению фактического содержания этилового спирта в параметре «alcoholVolume» фактическое содержание этилового спирта для продукции, произведенной на

территории Российской Федерации, дополнительно необходимо указывать сведения о первичном учетном документе, подтверждающем факт производства: в параметре «documentNumber» и в параметре «documentDate» дата документа.

- 8. Планируется запуск новой товарной группы «Пиротехнические изделия» (API V3):
  - a. Атрибутивный состав заказа будет содержать базовые объекты «Order» и «OrderProduct» включая расширение объекта «Order»:
    - i. Параметр «cisType» (Тип кода маркировки) объекта «OrderProduct» поддерживает допустимые значения: UNIT, GROUP, SET;
    - ii. Параметр «releaseMethodType» (Способ выпуска товаров в оборот), передаваемый в атрибуте «attributes» объекта «Order», поддерживает допустимые значения: PRODUCTION (Производство в РФ), IMPORT (Ввезен в РФ);
    - iii. Параметр «contactPerson» (Контактное лицо) объекта «Order» опциональное значение;
    - iv. Параметр «createMethodType» (Способ изготовления СИ): валидация на соответствие значениям справочника снята;
    - v. Параметр «productionOrderId» (Идентификатор производственного заказа) — опциональное значение.
  - b. Атрибутивный состав отчёта о нанесении будет содержать базовый объект «UtilisationReport». В расширении объекта могут быть указаны опциональные атрибуты:
    - i. «productionDate» (Дата производства продукции) [0..1] поддерживаемый формат yyyy-MM-dd;
    - ii. «expirationDate» (Дата истечения срока годности) [0..1] поддерживаемый формат yyyy-MM-dd.
  - c. Условия заполнения опциональных параметров: в отчёте об использовании (нанесении) КМ должна быть указана дата срока годности продукции в атрибуте «expirationDate» и дата производства в атрибуте «productionDate» для единиц товара. Для групповых упаковок атрибуты «expirationDate» и «productionDate» могут не заполняться.

- Обработка отчета проводится асинхронно. Результат обработки будет содержаться в результирующей квитанции.
- d. Для эмиссии КМ доступно использование двух шаблонов кодов маркировки:
  - i. GS1\_AISTR\_SHORT AI01 14 AI21 6 <GS> AI93 4;
  - ii. GS1\_AISTR\_ASYM\_SHORT AI01 14 AI21 13 <GS> AI91 4 <GS> AI92 44.
- 9. Планируется запуск новой товарной группы «Очки и линзы» (API V3):
  - a. Атрибутивный состав заказа будет содержать базовые объекты «Order» и «OrderProduct» включая расширение объекта «Order»:
    - i. Параметр «cisType» (Тип кода маркировки) объекта «OrderProduct» поддерживает допустимые значения: UNIT, GROUP, SET;
    - ii. Параметр «releaseMethodType» (Способ выпуска товаров в оборот), передаваемый в атрибуте «attributes» объекта «Order», поддерживает допустимые значения: PRODUCTION (Производство в РФ), IMPORT (Ввезен в РФ);
    - iii. Параметр «contactPerson» (Контактное лицо) объекта «Order» опциональное значение;
    - iv. Параметр «createMethodType» (Способ изготовления СИ): валидация на соответствие значениям справочника снята;
    - v. Параметр «productionOrderId» (Идентификатор производственного заказа) — опциональное значение.
  - b. Атрибутивный состав отчёта о нанесении будет содержать базовый объект «UtilisationReport». В расширении объекта могут быть указаны опциональные атрибуты:
    - i. «productionDate» (Дата производства продукции) [0..1] поддерживаемый формат yyyy-MM-dd;
    - ii. «expirationDate» (Дата истечения срока годности) [0..1] поддерживаемый формат yyyy-MM-dd.
  - c. Условия заполнения опциональных параметров: в отчёте об использовании (нанесении) КМ должна быть указана дата срока годности продукции в атрибуте «expirationDate» и дата производства в атрибуте «productionDate» для единиц товара. Для групповых упаковок

- атрибуты «expirationDate» и «productionDate» могут не заполняться. Обработка отчета проводится асинхронно. Результат обработки будет содержаться в результирующей квитанции.
- d. Для эмиссии КМ доступно использование шаблона кодов маркировки:
  - i. GS1\_AISTR\_ASYM\_SHORT AI01 14 AI21 13 <GS> AI91 4 <GS> AI92 44.
- e. В период с 01.06.2026 по 28.02.2027 планируется проведение эксперимента.
- 10.Для товарной группы «Радиоэлектронная продукция» будет реализовано закрытие заказов, созданных до 01.05.2026 с типом оплаты «Оплата по нанесению» со способом ввода в оборот REAPPLY.
- 11.Отменяется переход на использование идентификаторов UUID с версии 4 на версию 8 (согласно RFC – 9562).
