## <span id="page-4-2"></span>**▪️ Отгрузка безалкогольного пива, а также пива и слабоалкогольных напитков в Республику Казахстан**

#### • **Затронутые товарные группы**

- Безалкогольное пиво (22 / nabeer)
- Пиво, напитки, изготавливаемые на основе пива, слабоалкогольные напитки (15 / beer)

#### • **Бизнес-контекст**

Будет реализован новый процесс экспорта в режиме признания кодов маркировки в Республику Казахстан:

- 1. Маркированный товар передаётся экспортёром на территории РФ с помощью документа «Отгрузка в ЕАЭС» (EAS\_CROSSBORDER\_EXPORT) в формате \* .json.
- 2. Импортёр из страны ЕАЭС подаёт документ приёмки, который содержит сведения о принимаемых товарах, указанных в документе отгрузки. После успешной обработки документа принятые коды маркировки становятся доступны для оборота на территории страны ЕАЭС, а экспортёру из РФ для просмотра доступен документ «Приёмка в ЕАЭС» (EAS\_CROSSBORDER\_EXPORT\_ACCEPTANCE).

#### • **Изменение в True API**

✅ Откроется документ «Отгрузка в ЕАЭС» (EAS\_CROSSBORDER\_EXPORT)

## <span id="page-10-0"></span>**▪️ Упрощение процесса импорта ноутбуков и смартфонов из Республики Армения**

#### • **Затронутые товарные группы**

◦ Радиоэлектронная продукция. Ноутбуки и смартфоны (51 / gadgets)

#### • **Бизнес-контекст**

Планируется реализация упрощённого процесса импорта в Российскую Федерацию из Республики Армения.

Будет реализована следующая схема для импорта товаров по упрощённому процессу:

- 1. Экспортёр из Республики Армения описывает товар в Национальном каталоге и заказывает российские коды маркировки через Систему маркировки Республики Армения.
- 2. Далее экспортёр наносит коды маркировки на товары и подаёт в Систему маркировки Республики Армения сведения о трансграничном перемещении товаров. Эти сведения импортёр сможет посмотреть в Системе маркировки РФ в документе «Отгрузка из ЕАЭС» (LP\_SHIP\_GOODS\_CROSSBORDER).
- 3. Импортёр подаёт в Систему маркировки РФ документ «Приёмка» (LP\_ACCEPT\_GOODS), который содержит сведения о приёмке товаров, указанных в документе отгрузки. После успешной обработки документа приёмки принятые коды маркировки будут введены в оборот.

#### • **Изменение в True API**

✅ Откроется документ: «Приёмка отгрузки из ЕАЭС» (LP\_ACCEPT\_GOODS) на документ «Отгрузка из ЕАЭС» (LP\_SHIP\_GOODS\_CROSSBORDER)

#### • **Подробности**

Требования к заполнению параметров в документе «Приёмка отгрузки из ЕАЭС» (LP\_ACCEPT\_GOODS) на документ «Отгрузка из ЕАЭС» (LP\_SHIP\_GOODS\_CROSSBORDER) будут аналогичны требованиям, применимым при импорте *ноутбуков и смартфонов* из Республики Беларусь.

Описание документов см. в [True API.](https://docs.crpt.ru/gismt/True_API/)

## <span id="page-20-0"></span>**▪️ Импорт консервов и морепродуктов в режиме взаимного признания кодов идентификации из Республики Беларусь**

#### • **Затронутые товарные группы**

- Консервированная продукция (32 / conserve)
- Морепродукты (21 / seafood)

#### • **Бизнес-контекст**

Будет доступен импорт из Республики Беларусь в режиме взаимного признания кодов идентификации. Это значит, что Система маркировки будет обеспечивать получение, обработку и признание кодов маркировки, эмитированных в Системе маркировки Республики Беларусь, указанных в документе трансграничной отгрузки, полученном от Оператора Республики Беларусь.

Импорт будет реализован следующим образом:

- 1. Экспортёр из ЕАЭС формирует документ «Отгрузка из ЕАЭС с признанием КИ» (EAS\_CROSSBORDER), в котором указывает передаваемые коды маркировки и их атрибуты.
- 2. Импортёр в РФ на основании полученного документа отгрузки формирует документ «Приёмка отгрузки из ЕАЭС» (LP\_ACCEPT\_GOODS) в одном из следующих вариантов:
  - полная приёмка;
  - частичная приёмка;
  - отказ от приёмки.
- 3. Импортёр в РФ может отредактировать в документе «Приёмка отгрузки из ЕАЭС» (LP\_ACCEPT\_GOODS) атрибуты по каждому коду маркировки, переданному в документе отгрузки.

При успешной обработке документа «Приёмка отгрузки из ЕАЭС» (LP\_ACCEPT\_GOODS) импортёр становится владельцем кодов маркировки, а статус кодов меняется на В обороте.

#### • **Изменение в True API**

✅ Откроется документ: «Приёмка отгрузки из ЕАЭС» (LP\_ACCEPT\_GOODS) на документ «Отгрузка из ЕАЭС с признанием КИ» (EAS\_CROSSBORDER)

#### • **Подробности**

Требования к заполнению параметров в документе «Приёмка отгрузки из ЕАЭС» (LP\_ACCEPT\_GOODS) на документ «Отгрузка из ЕАЭС с признанием КИ» (EAS\_CROSSBORDER) аналогичны требованиям, применимым при импорте товаров из Республики Армения.

## <span id="page-21-0"></span>**▪️ Экспорт консервов и морепродуктов в режиме взаимного признания кодов идентификации в Республику Беларусь**

#### • **Затронутые товарные группы**

- Консервированная продукция (32 / conserve)
- Морепродукты (21 / seafood)

#### • **Бизнес-контекст**

Будет реализован новый процесс экспорта в режиме взаимного признания кодов маркировки в Республику Беларусь:

- 1. Маркированный товар передаётся экспортёром на территории РФ с помощью документа «Отгрузка в ЕАЭС» (EAS\_CROSSBORDER\_EXPORT) в формате \* .json.
- 2. Импортёр из страны ЕАЭС подаёт документ приёмки, который содержит сведения о принимаемых товарах, указанных в документе отгрузки. После успешной обработки документа принятые коды маркировки становятся доступны для оборота на территории страны ЕАЭС, а экспортёру из РФ для просмотра доступен документ «Приёмка в ЕАЭС» (EAS\_CROSSBORDER\_EXPORT\_ACCEPTANCE).

#### • **Изменение в True API**

## <span id="page-35-0"></span>**▪️ Импорт БАД, ветпрепаратов, духов и туалетной воды, косметики, бытовой химии и товаров личной гигиены в режиме взаимного признания кодов идентификации из Республики Армения**

## <span id="page-42-0"></span>**▪️ Работа с обращениями потребителей**

#### • **Затронутые товарные группы**

#### • **Бизнес-контекст**

- получать список обращений, касающихся ваших товаров;
- ознакомляться с деталями: описанием, фото, данными о товаре и торговой точке;
- отправлять официальные ответы с пояснением и прикреплёнными фото.

Как это будет работать:

- 1. Потребитель обнаруживает признак нарушения например, брак, просрочку или ошибку в маркировке — и отправляет обращение в мобильном приложении «Честный ЗНАК».
- 2. Система определяет, с каким участником оборота заключено соглашение, и перенаправляет ему обращение.
- 3. Данный участник оборота товаров изучает обращение, проверяет ситуацию и направляет ответ.
- 4. Ответ проходит модерацию со стороны Оператора:
  - если ответ конструктивен и по существу потребитель видит ответ в приложении;
  - если ответ не содержит пояснений или уклоняется от темы обращение передаётся в органы госвласти (например, Роспотребнадзор).

Благодаря такой возможности, вы становитесь первым звеном рассмотрения обращений, что даёт важные преимущества:

- быстрее решать вопросы с качеством, маркировкой и условиями продажи;
- снижать риски выявлять проблемы в цепочке поставок на раннем этапе;
- быстрее закрывать претензии без участия госорганов;
- защищать репутацию оперативно реагировать и показывать свою позицию;
- укреплять доверие демонстрировать прозрачность и заботу о потребителе;
- получать аналитику видеть, сколько обращений, по каким товарам и в каких регионах.

Доступ к функциональности будет предоставляться после заключения соглашения на работу с обращениями. Запрос на подключение можно будет направить в службу технической поддержки после запуска функциональности.

#### • **Подробности**
