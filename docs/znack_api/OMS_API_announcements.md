# OMS-CLOUD 4.0 ANNOUNCEMENT OF PLANNED CHANGES IN THE OMS Main functions Revision 1.26

Sheets 26

#### **ABSTRACT**

This document is intended to inform in advance the GCPs on the changes planned for implementation in the OMS API in the next 1-2 months. The changes described are for informational purposes only and are published in advance only to enable the GCP to plan the integration works with the OMS.

## <span id="page-3-0"></span>**1. ANNOUNCEMENT OF PLANNED CHANGES IN OMS API**

This section provides a list of planned modifications (which are at the stage of analysis) in the OMS API.

- 1. It is planned to disable support of the attached signature. Dates for disabling will be announced later.
- 2. The createMethodType parameter in the order structure will be deprecated across all system interfaces (API V3, SHC API, EEU API). The parameter will become optional, and its value will no longer be validated. The order structure will be processed correctly with or without the createMethodType parameter.
- 3. For the "Specialized food products and biologically active food additives" goods group, marking code emission will be enabled for goods in the new fourth payment wave starting March 31, 2026.
- 4. A transition from UUID v4 to v8 (according to RFC 9562) is planned due to changes in the internal UUID generation logic in the OMS. After the transition to UUID v8, the string format of UUIDs will remain unchanged. Generation of new UUID v8 identifiers will start in the production environment on September 1, 2026.
- 5. Starting April 29, 2026, you will be able to submit information on MCs with variable expiration dates for the "Confectionery products" goods group in the application report using the "expDateConditions" parameter (Product expiration date (shelf life exceeding 72 hours) with indication of storage condition index). This parameter contains a set of expiration date values. Each item in the set must include:
  - a storage condition index specified in the NCMG, and
  - an expiration date (must be greater than or equal to the current date) in yyyy-MM-dd format.

Several expiration dates can be provided. Each storage condition index must be unique within the set.

6. From April 29, 2026, to March 1, 2027, goods marking for the "Medical products" goods group will be carried out as part of a pilot program for GTINs in the eighth payment group.

- 7. For the "Dairy products" goods group, emission will be available for the fifth payment group. Starting July 1, 2026, the "Payment upon emission" payment type will be available.
- 8. For the "Beer, beer-based and low-alcohol beverages" goods group, specifying the actual ethyl alcohol content in the "alcoholVolume" parameter (actual content of ethyl alcohol for goods manufactured in the Russian Federation) will become mandatory; it will also be necessary to provide details of the primary accounting document confirming production in the "documentNumber" and "documentDate" parameters.
- 9. It is planned to launch a new goods group "Pyrotechnic articles" (API V3):
  - a. The order attribute set will include the "Order" and "OrderProduct" base objects, including the "Order" object extension:
    - i. The "cisType" parameter (Marking code type) of the "OrderProduct" object supports the following values: UNIT, GROUP, SET;
    - ii. The "releaseMethodType" parameter (Method of goods introduction into circulation) transferred in the "attributes" attribute of the "Order" object supports the following values: PRODUCTION (Manufacture in the Russian Federation), IMPORT (Imported into the Russian Federation);
    - iii. The "contactPerson" parameter (Contact person) of the "Order" object is optional;
    - iv. The "createMethodType" parameter (IM creation method): validation against guide values has been removed;
    - v. The "productionOrderId" parameter (Production order identifier) is optional.
  - b. The application report attribute set will include the "UtilisationReport" base object. The object extension may contain the following optional attributes:
    - i. "productionDate" (Production date) [0..1] supported format: yyyy-MM-dd;
    - ii. "expirationDate" (Expiration date) [0..1] supported format: yyyy-MMdd.
  - c. Requirements for optional parameters: in the report on MC utilization (application), the product expiration date must be specified in the "expirationDate" attribute and the production date must be specified in the

"productionDate" attribute for units of goods. For group packages, the "expirationDate" and "productionDate" attributes may be unfilled. The report is processed asynchronously. The processing result will be in the result receipt.

- d. Two marking code templates are available for the MC emission:
  - i. GS1\_AISTR\_SHORT AI01 14 AI21 6 <GS> AI93 4;
  - ii. GS1\_AISTR\_ASYM\_SHORT AI01 14 AI21 13 <GS> AI91 4 <GS> AI92 44.
- 10.It is planned to launch a new goods group "Eyeglasses and lenses" (API V3):
  - a. The order attribute set will include the "Order" and "OrderProduct" base objects, including the "Order" object extension:
    - i. The "cisType" parameter (Marking code type) of the "OrderProduct" object supports the following values: UNIT, GROUP, SET;
    - ii. The "releaseMethodType" parameter (Method of goods introduction into circulation) transferred in the "attributes" attribute of the "Order" object supports the following values: PRODUCTION (Manufacture in the Russian Federation), IMPORT (Imported into the Russian Federation);
    - iii. The "contactPerson" parameter (Contact person) of the "Order" object is optional;
    - iv. The "createMethodType" parameter (IM creation method): validation against guide values has been removed;
    - v. The "productionOrderId" parameter (Production order identifier) is optional.
  - b. The application report attribute set will include the "UtilisationReport" base object. The object extension may contain the following optional attributes:
    - i. "productionDate" (Production date) [0..1] supported format: yyyy-MM-dd;
    - ii. "expirationDate" (Expiration date) [0..1] supported format: yyyy-MMdd.
  - c. Requirements for optional parameters: in the report on MC utilization (application), the product expiration date must be specified in the "expirationDate" attribute and the production date must be specified in the "productionDate" attribute for units of goods. For group packages, the

"expirationDate" and "productionDate" attributes may be unfilled. The report is processed asynchronously. The processing result will be in the result receipt.

- d. The following marking code template is available for the MC emission:
  - i. GS1\_AISTR\_ASYM\_SHORT AI01 14 AI21 13 <GS> AI91 4 <GS> AI92 44.
- e. The experimental period is planned from June 1, 2026, to February 28, 2027.
- 11.For the "Radio-electronic products" goods group, orders created before May 1, 2026, with the "Payment upon application" payment type and the method of introduction into circulation REAPPLY will be closed.
- 12.The transition from UUID version 4 to version 8 identifiers (according to RFC 9562) is cancelled.
